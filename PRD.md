## 1. 产品定位

### 1.1 一句话定义
**面向业余爱好者**的智能互动音乐教学 App，覆盖 **7 种乐器**（吉他、尤克里里、钢琴、拇指琴、口琴、古筝、非洲鼓），主打"乐音识别 + 曲谱库 + 名师课 + 智能陪练 + 学习社区"五大能力。**自研版目标：完整复刻功能 + 全部免费/低成本 + 支持本地/自托管**。

## 3. 核心业务流程

### 3.1 典型学习闭环（与原版一致）
```
打开 App → 首页推荐/选课
   ↓
[新手]测评（可选）→ 选乐器 → 选课程
   ↓
看谱 + 听示范音频 → 调速 → 跟弹
   ↓
AI 评分 → 错音回放 → 标记完成
   ↓
解锁下一关 / 累计学习天数 → 升级勋章
   ↓
发布到"发现区" / 收藏 / 分享
```

### 3.2 用户痛点驱动（用户当前诉求：高阶练习）
原版**最大的槽点**（社区反馈推测）：会员费用高、付费墙深。
自研版针对性：
- **不设付费墙**：所有课程、曲库、评分报告**全部免费**
- **AI 高阶陪练**：弱项诊断 + 专项训练包
- **AI 扒谱**：任意歌曲上传即可生成跟弹谱

## 4. AI 引擎能力拆解（端侧 + 云端协同）

| 能力 | 端侧 / 云端 | 候选方案 | 优先级 |
| --- | --- | --- | --- |
| **音高识别** | **端侧** | CREPE / SPICE（轻量 CNN） | **P0** |
| **节拍识别** | **端侧** | librosa beat tracker + DTW | **P0** |
| **和弦识别** | 云端 | chroma + HMM / Basic Pitch | P0 |
| **评分模型** | 云端 | 加权规则 + 可学习参数 | P0 |
| **扒谱** | 云端 | Basic Pitch / 自训练 | P1 |
| **推荐** | 云端 | 协同过滤 + 规则 | P1 |
| **歌词对齐** | 云端 | Whisper + forced alignment | P2 |

### 4.1 端侧技术栈（用 context7 已查最新版）
- **Flutter + flutter_audio_capture** 或 **native AudioRecord**
- **onnxruntime-mobile** 跑 CREPE 模型
- **TFLite** 备选

### 4.2 云端技术栈
- Python + FastAPI
- librosa + soundfile
- PostgreSQL + Redis
- FFmpeg 音频处理

## 12. v0.3 新增：智能规划 Agent 平台

> 本节是 v0.3 相对 v0.2 的核心增量。

### 12.1 背景与动机

App 本身的开发已交付基础功能（T01-T03 完成）。但您作为独立开发者，面临的核心挑战是：

> **如何高效拆解任务、协调多个 Agent 并行工作、最终整合成可用产品？**

传统的"线性开发"模式无法应对 24+ 个 P0/P1 任务的规模。需要一个**跨任务的规划与编排平台**。

### 12.2 平台定位

**`ukulele-agent`** —— 一个**项目无关的可移植规则与编排平台**：

```
                ┌──────────────────────────┐
                │   ukulele-agent 平台      │
                │  ┌─────────────────────┐ │
                │  │ Plan Engine (拆任务) │ │
                │  │ Rule Engine (规则库) │ │
                │  │ Orchestrator (调度)  │ │
                │  └─────────────────────┘ │
                └──────────────────────────┘
                       ↑             ↓
                  Web Dashboard   GitHub API
                  (管理+监控)     (创建 PR)
                       ↑             ↓
              ┌─────────────────────────────────┐
              │  任何项目（ukulele / 其他）       │
              │  通过 GitHub Issue + PR 接入      │
              └─────────────────────────────────┘
```

### 12.3 核心能力

| 能力 | 描述 | 用户价值 |
| --- | --- | --- |
| **目标拆解** | 输入自然语言目标 → 输出 DAG 任务图 | 不再手写 todo |
| **规则可移植** | 规则存为 YAML，可导出给其他项目 | 一次编写，多处使用 |
| **多 Agent 协作** | Claude Opus/Sonnet/Haiku + 本地脚本 + 人工 | 灵活调度 |
| **GitHub 原生集成** | 每 task 创建分支+PR，merge 自动完成 | 复用 GitHub 工作流 |
| **Web 监控** | 实时看任务进度、Token 用量、审计日志 | 透明可控 |
| **失败重试** | 单任务最多 3 次自动重试 + 切换模型 | 鲁棒性 |

### 12.4 与 App 的边界

| 维度 | ukulele App | ukulele-agent |
| --- | --- | --- |
| 用户 | 音乐学习者 | 开发者 / 产品经理 |
| 业务 | 调音、跟弹、评分 | 拆任务、调度、审计 |
| AI 用途 | 乐音识别（实时） | 任务编排（异步） |
| 数据 | 学习记录 | 计划、任务、规则 |
| 部署 | Flutter App + 后端 | 独立 FastAPI + Web（可复用数据库） |

### 12.5 商业化潜力

虽然 MVP 全免费，但 Agent 平台本身可作为**对外服务**：

- **SaaS**：开发者订阅（按 token 用量计费）
- **私有部署**：企业内网独立部署
- **开源**：核心开源，高级功能付费

### 12.6 与原版 App 的差异化

原版 AI 音乐学园是**单一终端 App**。我们的差异化：

1. **双层产品**：App（面向用户）+ Agent 平台（面向开发者）
2. **开放生态**：规则可移植、平台可独立部署
3. **本地优先**：规则与数据本地可控，避免云绑死
4. **AI Native**：从开发到运行全程 AI 驱动
